# frappe_mobile_item_app

Production-grade Flutter mobile app for Frappe/ERPNext Item CRUD with clean architecture.

## Implemented Decisions
- First-run login (Base URL + API Key/Secret) with secure storage
- ERPNext standard doctypes (Item, Item Group, UOM)
- Fields: required 5 + `disabled`, `has_variants`, `valuation_rate`
- `item_code` auto-generated by backend (client treats as optional)
- Infinite scroll pagination with debounce search
- Soft delete + hard delete flow (permission handled by backend)
- Single account per device
- Read-cache offline fallback via Drift
- Riverpod + go_router + Dio + json_serializable + drift
- Mobile-first UI with tablet split-pane adaptation

## Architecture
- `lib/core`: network, interceptors, storage, logging, failures
- `lib/features/auth`: login/session flow
- `lib/features/items`: item domain/data/presentation + CRUD
- `lib/features/shell`: app drawer shell
- `lib/app`: theme, router, bootstrap
